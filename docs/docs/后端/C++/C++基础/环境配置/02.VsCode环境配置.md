## 2.MSYS2

在`MinGW-64`官网中，如果我们往下看，可以看到除了`MinGW-64`的`github`地址，还有`MSYS2`和`w64devkit`

![image-20260630155716035](https://gitee.com/xarzhi/picture/raw/master/img/image-20260630155716035.png)

那么这都是什么东西呢

- **MinGW-w64**：编译器工具链（项目本身）
  - MinGW-w64 是一个**开源项目**，提供 Windows 平台的 GNU 工具链头文件、导入库和运行时，让 GCC/Clang 能在 Windows 上编译出**原生 Windows 程序（.exe，不依赖 Cygwin DLL）**，支持 32 位和 64 位
- **w64devkit**：MinGW-w64 的极简便携发行版
  - w64devkit（由 skeeto 维护）是**一个精心打包好的 MinGW-w64 + GCC 工具链压缩包**，解压即用：
    - 内含：MinGW-w64 GCC（gcc/g++）、GDB、GNU Make、busybox（提供 sh/ls 等 Unix 命令）、Vim、Ccache
    - **零安装**：绿色软件，不写注册表，删文件夹即卸载
    - 适合：快速搭环境编译 C/C++、CI 构建、U盘随身携带
- **MSYS2**：带包管理器的类 Unix 开发平台
  - 提供 **bash shell**、git、autotools 等 Unix 工具
  - 通过 `pacman`一键安装/更新 GCC、Clang、CMake、FFmpeg、OpenSSL 等数千个包
  - 包含多个子环境（可共存）：
    - **UCRT64**（推荐新项目）：x86_64 + UCRT + GCC 
    - **MINGW64/MINGW32**：传统 MSVCRT 运行时 
    - **CLANG64**：Clang 编译器 
    - **MSYS**：运行依赖 msys-2.0.dll 的 Unix 工具（用于构建脚本，不用于最终程序）

#### 5.1.2 终端编码格式

在终端中输入如下指令

```bash

```

![image-20260630215729202](https://gitee.com/xarzhi/picture/raw/master/img/image-20260630215729202.png)

可以看到`Active code page: 65001`，那么`65001`就是`UTF-8`的字符编码编号

对应的GBK字符编码编号是`936`



### 5.2 乱码原因

乱码产生的主要原因是，**终端的编码格式和编译器代码的编码格式不一样**

有以下四种情况

| 代码编码方式 | 终端编码方式 | 是否统一 |
| ------------ | ------------ | -------- |
| GBK          | GBK          | 统一     |
| UTF-8        | UFT-8        | 统一     |
| GBK          | UFT-8        | 不统一   |
| UTF-8        | GBK          | 不统一   |

所以，想要程序执行不乱码，只要代码编码方式和终端编码方式统一就行

不管是统一为`UTF-8`还是统一为`GBK`

不过`UTF-8`是国际通用的字符编码格式，还是推荐`UTF-8`

:::tip **GBK**和**UTF-8**

UTF-8

- 优点：
  - 国际通用支持所有语言、emoj、特殊符号，避免乱码。
  - 跨平台：Linux、macOS、Web、现代编程语言默认都使用UTF-8。
  - 未来兼容：Windows10/11已逐步将UTF-8设为系统默认（需开启Beta功能）。
  - 无编码冲突：比如JSON、XML、HTML推荐用UTF-8，GBK可能导致解析错误。
- 缺点：主要在旧版Windows命令行时
  - 少数老Windows工具(如find、sort）在代码页65001下可能行为异常。
  - 某些仅支持ANSI的老程序（如用C标准/O未处理Unicode的）输出会乱码。



GBK

- 优点
  - 与简体中文Windows原生ANSI完美兼容（记事本保存为ANSI就是GBK）。
  - 某些老行业软件（银行、税务、政府旧系统）强制要求GBK.
  - Windows命令行中老工具运行最稳定。
- 缺点
  - 不是国际标准，其他系统默认不识别
  - 无法表示生僻字、emoji、多语言混合文本。
  - 跨平台传输（比如代码上传到Git）容易变成乱码。

:::

#### 5.3.2 修改终端编码格式

修改终端编码格式需要在电脑的设置里修改

打开电脑设置，`win+i`快捷键可直接打开

点击时间和语言→语言和区域

![image-20260630221328511](https://gitee.com/xarzhi/picture/raw/master/img/image-20260630221328511.png)

往下滑找到管理语言设置打开

![image-20260630221428015](https://gitee.com/xarzhi/picture/raw/master/img/image-20260630221428015.png)

点击更改系统区域设置

勾选Beta版：....UniCode UTf-8......

然后确定

确定之后，会让你系统重启，重启之后就可以了

![image-20260630221457934](https://gitee.com/xarzhi/picture/raw/master/img/image-20260630221457934.png)



之后再执行，就不会乱码了
