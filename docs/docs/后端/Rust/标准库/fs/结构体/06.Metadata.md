# Metadata

> Struct std::fs::Metadata

有关文件的元数据信息。

此结构体是从 [`metadata`](../index.md#metadata) 或 [`symlink_metadata`](../index.md#symlink_metadata) 函数或方法返回的，表示有关文件的已知元数据，例如其权限，大小，修改时间等。



## 1.Implementations

### Impl Metadata

#### file_type

返回此元数据的文件类型

```rust
pub fn file_type(&self) -> FileType
```

**返回值**：返回一个[`FileType`](./05.FileType)

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();

    println!("{:#?}", metadata.file_type());
	/*
	FileType {
		is_file: true,
		is_dir: false,
		is_symlink: false,
		..
	}
	 */
}

```



#### is_dir

判断是否为目录（文件夹），并且对于从 [`symlink_metadata`](https://www.rustwiki.org.cn/zh-CN/std/fs/fn.symlink_metadata.html) 获得的符号链接元数据将为 false。

```rust
pub fn is_dir(&self) -> bool
```

**返回值**：返回`bool`值，判断self是否为目录

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("fold").unwrap();

    println!("{:#?}", metadata.is_dir());  // true
}
```



#### is_file

判断是否为文件，并且对于从 [`symlink_metadata`](https://www.rustwiki.org.cn/zh-CN/std/fs/fn.symlink_metadata.html) 获得的符号链接元数据将为 false。

```rust
pub fn is_file(&self) -> bool
```

**返回值**：返回`bool`值，判断self是否为文件

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();

    println!("{:#?}", metadata.is_file());   // true
}
```

:::tip

当目标只是读取 (或写入) 源时，可以读取 (或写入) 最可靠的测试源方法是打开它。 例如，仅使用 `is_file` 才能中断类似 Unix 的系统上的工作流，例如 `diff <( prog_a )`。 有关更多信息，请参见 [`File::open`](https://www.rustwiki.org.cn/zh-CN/std/fs/struct.File.html#method.open) 或 [`OpenOptions::open`](https://www.rustwiki.org.cn/zh-CN/std/fs/struct.OpenOptions.html#method.open)。

:::

#### permissions

获取文件拥有什么权限

```rust
pub fn permissions(&self) -> Permissions
```

**返回值**：返回一个[`Permissions`](./08.Permissions)结构体对象，可根据这个对象获取和设置文件只读。

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();
	let mut p=metadata.permissions();

    println!("{:#?}", p.readonly()); // false

	p.set_readonly(true);

    println!("{:#?}", p.readonly()); // true
}
```



#### modified

获取文件最后的修改时间

```rust
pub fn modified(&self) -> Result<SystemTime>
```

**返回值**：返回一个`Result`，若获取成功则为`Ok(SystemTime)`，否则返回相应错误信息

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();

    println!("{:#?}", metadata.modified()); 
	/*
		Ok(
			SystemTime {
				intervals: 134221820054135173,
			},
		)
	 */
}
```

:::tip

返回的值对应于 Unix 平台上的 `stat` 的 `mtime` 字段和 Windows 平台上的 `ftLastWriteTime` 字段。

此字段可能并非在所有平台上都可用，并且在它不可用的平台上将返回 `Err`。

:::



#### accessed

获取文件最后的访问时间

```rust
pub fn accessed(&self) -> Result<SystemTime>
```

**返回值**：返回一个`Result`，若获取成功则为`Ok(SystemTime)`，否则返回相应错误信息

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();

    println!("{:#?}", metadata.accessed()); 
	/*
        Ok(
            SystemTime {
                intervals: 134221820054135173,
            },
        )
	 */
}
```

:::tip

返回的值对应于 Unix 平台上的 `stat` 的 `atime` 字段和 Windows 平台上的 `ftLastAccessTime` 字段。

请注意，并非所有平台都会在文件的元数据中保留此字段的更新，例如，Windows 可以选择在访问文件时禁用此更新，而 Linux 同样具有 `noatime`。

此字段可能并非在所有平台上都可用，并且在它不可用的平台上将返回 `Err`。

:::



#### created

获取文件的创建时间

```rust
pub fn created(&self) -> Result<SystemTime>
```

**返回值**：返回一个`Result`，若获取成功则为`Ok(SystemTime)`，否则返回相应错误信息

```rust
use std::fs;

fn main() {
    let metadata = fs::metadata("test.txt").unwrap();

    println!("{:#?}", metadata.created()); 
	/*
        Ok(
            SystemTime {
                intervals: 134221820054135173,
            },
        )
	 */
}
```

:::tip

返回的值与从 4.11 开始的 Linux 内核上的 `statx` 的 `btime` 字段，在其他 Unix 平台上的 `stat` 的 `birthtime` 字段以及在 Windows 平台上的 `ftCreationTime` 字段相对应。

此字段可能并非在所有平台上都可用，并且会在它不可用的平台或文件系统上返回 `Err`。

:::
