# Result

> std::io::Result

```rust
pub type Result<T> = Result<T, Error>;
```

I/O 操作的专用 [`Result`] 类型，简化了常用的`Result`，错误信息由`std::io::Error`自动处理，不需要我们单独写第二个泛型参数

[`std::io`](https://www.rustwiki.org.cn/zh-CN/std/io/index.html) 广泛使用此类型进行可能产生错误的任何操作。

通常使用这种 typedef 来避免直接写出 [`io::Error`](https://www.rustwiki.org.cn/zh-CN/std/io/struct.Error.html)，否则直接映射到 [`Result`](https://www.rustwiki.org.cn/zh-CN/std/result/enum.Result.html)。

通常的 Rust 样式是直接导入类型，而 [`Result`](https://www.rustwiki.org.cn/zh-CN/std/result/enum.Result.html) 的别名通常不是，以便于区分它们。 [`Result`](https://www.rustwiki.org.cn/zh-CN/std/result/enum.Result.html) 通常被假定为 [`std::result::Result`](https://www.rustwiki.org.cn/zh-CN/std/result/enum.Result.html)，因此这个别名的用户通常会使用 `io::Result` 而不是隐藏 [prelude](https://www.rustwiki.org.cn/zh-CN/std/prelude/index.html) 对 [`std::result::Result`](https://www.rustwiki.org.cn/zh-CN/std/result/enum.Result.html) 的导入。



## 1.示例

一个方便的函数，将 `io::Result` 冒泡给其调用者：

```rust
use std::io;

fn get_string() -> io::Result<String> {
    let mut buffer = String::new();

    io::stdin().read_line(&mut buffer)?;

    Ok(buffer)
}
```
