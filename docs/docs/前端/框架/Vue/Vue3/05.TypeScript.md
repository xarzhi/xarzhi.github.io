### 1.6.4 修饰符

**public** 默认修饰符，TypeScript中类中的成员默认为public

**private** 类的成员不能在类的外部访问，子类也不可以

**protected** 类的成员不能在类的外部访问，但是子类中可以访问。如果一个类的构造函数，修饰符为protected，那么此类只能被继承，无法实例化。

**readonly** 关键字`readonly`可以将实例的属性，设置为只读

```ts
					
	public  	 自身	子类	外部
	protected   自身	子类 
	private     自身


class Animal
{
    public a = 1
    protected b = 2
    private c = 3
}

class Dog extends Animal {

    readonly d = 4

    constructor() {
        super()
        console.log(1, this.a)
    }
}

const dog1 = new Dog
console.log(2, dog1.a)
console.log(3, dog1.d)
// dog1.d = 44
```

## 1.9 type类型别名

````ts
type Type1 = string; 							  // 基本类型

type Type2 = { a:numer, b: number}  // 对象类型
type Type21 = any[]
type Type22 = (data1:string)=>any

type Type5 = {a:number} & {b:number}  // 交叉
type Type6<T> = { value: T } 	  	  	// 泛型


// 实现

	eat(food: string): void
}
````

## 1.12 泛型 

**概念**：**泛型就是把类型当成参数**   

**泛型： <> 里写类型参数**，一般可以用 T 来表示。

### 1.12.1 基本使用

#### 在函数中使用泛型

**注意**：`<>`写在`()`前

```ts
// 定义
function test <T> (arg:T):T{
  console.log(arg);
  return arg;
}

// 调用，在<>中传入类型
test<number>(111);// 返回值是number类型的 111
test<string | boolean>('hahaha')//返回值是string类型的 hahaha
test<string | boolean>(true);//返回值是布尔类型的 true
```



需求限制字符串返回字符串

```ts
function print(arg:string):string {
    console.log(arg)
    return arg
}
```

可能会传数值

```ts
function print(arg:string | number):string | number {
    console.log(arg)
    return arg
}
```

不知道传啥

```ts
function print(arg:any):any {
    console.log(arg)
    return arg
}


const res:string = print(123)    容易出现BUG
```

泛型

```ts
function print<T>(arg:T):T {
    console.log(arg)
    return arg
}

print<nunber>(111)
print<string>('111')
```



定义函数、接口、类时，不预先指定类型，使用的时候再指定。

```ts

	let arr = []
```

#### 在接口中使用泛型

```ts
interface IObj<T> {
    // 键: 类型
```

### 1.12.2 索引签名

索引签名的思想是在只知道键和值类型的情况下对结构未知的对象进行类型划分。

**用处**：当你有一个对象 里面有很多键 ，一个个写比较麻烦，写核心的，其他带过

#### 对象

```ts
interface Obj {
     // 对象的键的类型都是string
	 [key: string]: number
}

const obj: Obj = {a:1,b:2}
```

使用

```ts
interface Iobj {
    a: number,
    b: number,
    // 描述对象剩余的键都是字符串
    // 描述对象剩余的类型都是任意类型
    [key: string]: any
}

const obj: Iobj = {
    // 键和值
    // 键值对
    // 索引和值
    a: 1,
    b: 2,
    c: 'sadda',
    d: 'fasdfsdf',
    // ..
}
```



#### 数组

```ts
interface Iarr {
    // 数组的键是下表，所以类型是number
    [index:number]: number
}
const arr: Iarr = [111,222,33]


// 使用泛型
interface Arr<T> {
	 [insert: number]: T
}
const arr: Arr<string> = ['a', 'b']
```
