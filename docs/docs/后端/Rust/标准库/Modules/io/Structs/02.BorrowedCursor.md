# Struct std::io::BorrowedCursor

```rust
pub struct BorrowedCursor<'a> { /* private fields */ }
```

[`BorrowedBuf`](https://www.rustwiki.org.cn/zh-CN/std/io/struct.BorrowedBuf.html) 未填充部分的可写视图。

提供对底层 `BorrowedBuf` 的已初始化和未初始化部分的访问。 可以使用 [`append`](https://www.rustwiki.org.cn/zh-CN/std/io/struct.BorrowedCursor.html#method.append) 将数据直接写入游标，也可以通过获取部分或全部游标的切片并写入切片来间接写入数据。

在间接情况下，调用者必须在写入后调用 [`advance`](https://www.rustwiki.org.cn/zh-CN/std/io/struct.BorrowedCursor.html#method.advance) 来通知游标已经写入了多少字节。

一旦数据被写入游标，它就成为底层 `BorrowedBuf` 的填充部分的一部分，并且不能再被游标访问或重写。 即，游标跟踪底层 `BorrowedBuf` 的未填充部分。

生命周期 `'a` 是底层缓冲区的生命周期的界限 (这意味着它是该缓冲区中数据的传递界限)。

reborrow

#### advance

通过断言 n 字节已被填充来前进游标。

```rust
pub unsafe fn advance(&mut self, n: usize) -> &mut Self
```

推进后，n 字节不再可以通过游标访问，只能通过底层缓冲区访问。 即，缓冲区的填充部分增加了 n 个元素，而其未填充部分 (以及该游标的容量) 缩小了 n 个元素。

:::tip Safety

调用者必须确保游标的前 n 字节已正确初始化。

:::







#### ensure_init

初始化游标中的所有字节。

```rust
pub fn ensure_init(&mut self) -> &mut Self
```





#### set_init

断言游标的前 n 个未填充字节已初始化。

```rust
pub unsafe fn set_init(&mut self, n: usize) -> &mut Self
```

BorrowedBuf 假定字节永远不会被取消初始化，因此当调用该方法时使用的字节数少于已知要初始化的字节数时，该方法什么也不做。

:::tip Safety

调用者必须确保缓冲区的前 n 个字节已经被初始化。

:::



#### append

将数据，追加，到游标，在其缓冲区中推进位置。

```rust
pub fn append(&mut self, buf: &[u8])
```

:::tip Panics

如果 `self.capacity()` 小于 `buf.len()`，则发生 `panic`。

:::
