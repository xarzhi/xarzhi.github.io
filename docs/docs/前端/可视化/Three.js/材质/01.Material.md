# Material

`Material` 是所有材质的基类，但不能直接使用 `Material` 来创建可见的对象

要让几何体在场景中可见，必须使用 `Material` 的具体子类，如 `MeshBasicMaterial`、`MeshStandardMaterial`、`MeshPhongMaterial` 等。

所有其他材质类型都继承了以下属性和方法（尽管它们可能具有不同的默认值）。

`Material` 类本身是一个抽象基类，它定义了材质的基本属性和方法，但不提供实际的渲染效果。因此，如果没有使用具体的材质子类，三维对象将无法正确渲染和显示。

如果你想自定义材质，通常需要扩展 `THREE.ShaderMaterial` 或者其他具体的材质类，但是依然需要具体的类来处理渲染和效果。

## 属性(Properties)

### .[alphaHash](https://threejs.org/docs/index.html#api/zh/materials/Material.alphaHash) : Boolean

启用alphaHash透明度，这是[.transparent](https://threejs.org/docs/index.html#api/zh/materials/Material.transparent)或[.alphaTest](https://threejs.org/docs/index.html#api/zh/materials/Material.alphaTest)的替代方案。如果不透明度低于随机阈值，则不会渲染材质。随机化会引入一些颗粒或噪点，但相较于传统的Alpha blend方式，避免了透明度引起的深度排序问题。使用TAARenderPass可以有效减少噪点。

### .[alphaTest](https://threejs.org/docs/index.html#api/zh/materials/Material.alphaTest) : Float

设置运行alphaTest时要使用的alpha值。如果不透明度低于此值，则不会渲染材质。默认值为**0**。

### .[alphaToCoverage](https://threejs.org/docs/index.html#api/zh/materials/Material.alphaToCoverage) : Boolean

启用 alpha 覆盖。 只能与启用 MSAA 的上下文一起使用（意味着在创建渲染器时将抗锯齿参数 **antialias** 设置为 `true`）。 启用此选项将平滑剪裁平面边缘和 alphaTest 剪辑边缘上的锯齿。 默认值为 `false`。

### .[blendAlpha](https://threejs.org/docs/index.html#api/zh/materials/Material.blendAlpha) : Float

表示恒定混合颜色的 alpha 值。 默认值为 `0`。
此属性仅在使用 [ConstantAlpha](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations) 或 [OneMinusConstantAlpha](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations) 自定义混合时有效。

### .[blendColor](https://threejs.org/docs/index.html#api/zh/materials/Material.blendColor) : [Color](https://threejs.org/docs/index.html#api/zh/math/Color)

表示恒定混合颜色的 RGB 值。 默认值为 `0x000000`。
此属性仅在使用 [ConstantColor](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations) 或 [OneMinusConstantColor](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations) 自定义混合时有效。

### .[blendDst](https://threejs.org/docs/index.html#api/zh/materials/Material.blendDst) : Integer

混合目标。默认值为[OneMinusSrcAlphaFactor](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。 目标因子所有可能的取值请参阅[constants](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。 必须将材质的blending设置为[CustomBlending](https://threejs.org/docs/index.html#api/zh/constants/Materials)才能生效。

### .[blendDstAlpha](https://threejs.org/docs/index.html#api/zh/materials/Material.blendDstAlpha) : Integer

[.blendDst](https://threejs.org/docs/index.html#api/zh/materials/Material.blendDst)的透明度。 默认值为 **null**.

### .[blendEquation](https://threejs.org/docs/index.html#api/zh/materials/Material.blendEquation) : Integer

使用混合时所采用的混合方程式。默认值为[AddEquation](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。 混合方程式所有可能的取值请参阅[constants](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。 必须将材质的blending设置为[CustomBlending](https://threejs.org/docs/index.html#api/zh/constants/Materials)才能生效。

### .[blendEquationAlpha](https://threejs.org/docs/index.html#api/zh/materials/Material.blendEquationAlpha) : Integer

[.blendEquation](https://threejs.org/docs/index.html#api/zh/materials/Material.blendEquation) 的透明度. 默认值为 **null**.

### .[blending](https://threejs.org/docs/index.html#api/zh/materials/Material.blending) : Blending

在使用此材质显示对象时要使用何种混合。
必须将其设置为[CustomBlending](https://threejs.org/docs/index.html#api/zh/constants/Materials)才能使用自定义blendSrc, blendDst 或者 [page:Constant blendEquation]。 混合模式所有可能的取值请参阅[constants](https://threejs.org/docs/index.html#api/zh/constants/Materials)。默认值为[NormalBlending](https://threejs.org/docs/index.html#api/zh/constants/Materials)。

### .[blendSrc](https://threejs.org/docs/index.html#api/zh/materials/Material.blendSrc) : Integer

混合源。默认值为[SrcAlphaFactor](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。 源因子所有可能的取值请参阅[constants](https://threejs.org/docs/index.html#api/zh/constants/CustomBlendingEquations)。
必须将材质的blending设置为[CustomBlending](https://threejs.org/docs/index.html#api/zh/constants/Materials)才能生效。

### .[blendSrcAlpha](https://threejs.org/docs/index.html#api/zh/materials/Material.blendSrcAlpha) : Integer

[.blendSrc](https://threejs.org/docs/index.html#api/zh/materials/Material.blendSrc)的透明度。 默认值为 **null**.

### .[clipIntersection](https://threejs.org/docs/index.html#api/zh/materials/Material.clipIntersection) : Boolean

更改剪裁平面的行为，以便仅剪切其交叉点，而不是它们的并集。默认值为 **false**。

### .[clippingPlanes](https://threejs.org/docs/index.html#api/zh/materials/Material.clippingPlanes) : Array

用户定义的剪裁平面，在世界空间中指定为THREE.Plane对象。这些平面适用于所有使用此材质的对象。空间中与平面的有符号距离为负的点被剪裁（未渲染）。 这需要[WebGLRenderer.localClippingEnabled](https://threejs.org/docs/index.html#api/zh/renderers/WebGLRenderer.localClippingEnabled)为**true**。 示例请参阅[WebGL / clipping /intersection](https://threejs.org/examples/#webgl_clipping_intersection)。默认值为 **null**。

### .[clipShadows](https://threejs.org/docs/index.html#api/zh/materials/Material.clipShadows) : Boolean

定义是否根据此材质上指定的剪裁平面剪切阴影。默认值为 **false**。

### .[colorWrite](https://threejs.org/docs/index.html#api/zh/materials/Material.colorWrite) : Boolean

是否渲染材质的颜色。 这可以与网格的renderOrder属性结合使用，以创建遮挡其他对象的不可见对象。默认值为**true**。

### .[defines](https://threejs.org/docs/index.html#api/zh/materials/Material.defines) : Object

注入shader的自定义对象。 以键值对形式的对象传递，{ MY_CUSTOM_DEFINE: '' , PI2: Math.PI * 2 }。 这些键值对在顶点和片元着色器中定义。默认值为**undefined**。

### .[depthFunc](https://threejs.org/docs/index.html#api/zh/materials/Material.depthFunc) : Integer

使用何种深度函数。默认为[LessEqualDepth](https://threejs.org/docs/index.html#api/zh/constants/Materials)。 深度模式所有可能的取值请查阅[constants](https://threejs.org/docs/index.html#api/zh/constants/Materials)。

### .[depthTest](https://threejs.org/docs/index.html#api/zh/materials/Material.depthTest) : Boolean

是否在渲染此材质时启用深度测试。默认为 **true**。

### .[depthWrite](https://threejs.org/docs/index.html#api/zh/materials/Material.depthWrite) : Boolean

渲染此材质是否对深度缓冲区有任何影响。默认为**true**。

在绘制2D叠加时，将多个事物分层在一起而不创建z-index时，禁用深度写入会很有用。

### .[forceSinglePass](https://threejs.org/docs/index.html#api/zh/materials/Material.forceSinglePass) : Boolean

决定双面透明的东西是否强制使用单通道渲染，默认为`false`。

为了减少一些半透明物体的渲染错误，此引擎调用两次绘制来渲染渲染双面透明的东西。 但是此方案可能会导致在某些情况下使绘制调用次数翻倍，例如渲染一些平面的植物例如草精灵之类的。 在这些情况下，将`forceSinglePass`设置为`true`来使用单通道渲染来避免性能问题。

### .[stencilWrite](https://threejs.org/docs/index.html#api/zh/materials/Material.stencilWrite) : Boolean

### .[customProgramCacheKey](https://threejs.org/docs/index.html#api/zh/materials/Material.customProgramCacheKey) () : String

当用到onBeforeCompile回调的时候，这个回调函数可以用来定义在onBeforeCompile中使用的配置项，这样three.js就可以根据这个回调返回的字符串来判定使用一个缓存的编译好的着色器代码还是根据需求重新编译一段新的着色器代码。

例如一个onBeforeCompile回调函数包含了下面的条件语句:
`if ( black ) { 	shader.fragmentShader = shader.fragmentShader.replace('gl_FragColor = vec4(1)', 'gl_FragColor = vec4(0)') }`那么 customProgramCacheKey 就可以设置为:
`material.customProgramCacheKey = function() { 	return black ? '1' : '0'; }`

和其他属性不一样的是，这个回调在[.clone](https://threejs.org/docs/index.html#api/zh/materials/Material.clone)()，[.copy](https://threejs.org/docs/index.html#api/zh/materials/Material.copy)() 和 [.toJSON](https://threejs.org/docs/index.html#api/zh/materials/Material.toJSON)() 中不支持。

values -- 具有参数的容器。 根据**values**设置属性。

meta -- 包含有元数据的对象，例如该对象的纹理或图片。 将material对象转换为 three.js [JSON Object/Scene format](https://github.com/mrdoob/three.js/wiki/JSON-Object-Scene-format-4)（three.js JSON 物体/场景格式）。
